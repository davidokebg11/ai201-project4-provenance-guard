# Provenance Guard

A backend API system for detecting whether submitted text content was written by a human or generated by an AI writing tool. Built for creative sharing platforms to provide transparent attribution labels and a fair appeals process.

---

## Architecture Overview

A submitted piece of text enters via `POST /submit` with a `text` field and a `creator_id`. It passes through two independent detection signals: (1) a Groq LLM classifier that holistically assesses whether the text reads as AI-generated, and (2) a stylometric heuristics analyzer that computes statistical properties of the writing. Both signal scores are combined into a single confidence score, which is mapped to a transparency label. The full result — including both signal scores, confidence, attribution, and label — is written to a structured JSON audit log and returned to the caller.

An appeal enters via `POST /appeal` with a `content_id` and `creator_reasoning`. The system looks up the original classification in the audit log, updates its status to `under_review`, appends the appeal reasoning, and returns a confirmation.

SUBMISSION FLOW:

POST /submit

├──► Signal 1: Groq LLM Classifier → llm_score (0–1)

├──► Signal 2: Stylometric Heuristics → stylo_score (0–1)

├──► Confidence Scorer → confidence (0–1) + attribution

├──► Transparency Labeler → label text

├──► Audit Log → structured JSON entry

└──► JSON Response
APPEAL FLOW:

POST /appeal

├──► Lookup content_id in audit log

├──► Update status → "under_review"

├──► Append appeal_reasoning

└──► JSON Response

## Detection Signals

### Signal 1: Groq LLM Classifier (`detector.py`)
Sends the submitted text to `llama-3.3-70b-versatile` via Groq with a prompt asking it to return a single float between 0 (human) and 1 (AI). This signal captures semantic and stylistic coherence holistically — things like unnaturally smooth transitions, hedging phrases, corporate-speak, and overly balanced structure that are hard to measure statistically.

**Why chosen:** LLMs are uniquely capable of detecting the "feel" of AI-generated text at a semantic level that pure statistics cannot capture.

**Blind spot:** Cannot reliably detect AI text that has been heavily edited by a human, or human text that is naturally formal and polished (e.g., academic writing).

### Signal 2: Stylometric Heuristics (`stylometrics.py`)
Computes three statistical properties of the text in pure Python:
- **Sentence length variance:** AI text tends to have more uniform sentence lengths (low standard deviation). Normalized against a typical human std_dev of ~15.
- **Type-token ratio (TTR):** Measures vocabulary diversity. AI text reuses words more; human writing is more varied.
- **Punctuation density:** Flags unusually low or high punctuation density as a mild AI indicator.

**Why chosen:** These structural properties are genuinely independent from the LLM signal — one is semantic, one is statistical. Together they are more informative than either alone.

**Blind spot:** Short texts (under ~50 words) give unreliable statistics. Also, consistent technical writers may score as AI-like.

## Confidence Scoring

Signals are combined with a weighted average — LLM score weighted 60%, stylometrics 40% — because the LLM signal is more semantically aware and generally more reliable:

confidence = (llm_score × 0.60) + (stylo_score × 0.40)

Thresholds:
- `0.00 – 0.39` → `likely_human`
- `0.40 – 0.64` → `uncertain`
- `0.65 – 1.00` → `likely_ai`

The uncertain band is intentionally wide (0.40–0.64) to reduce false positives. Mislabeling a human's work as AI is worse than missing an AI submission.

### Example Submissions

**High-confidence AI (corporate AI paragraph):**
```json
{
  "attribution": "likely_ai",
  "confidence": 0.6771,
  "llm_score": 0.9,
  "stylo_score": 0.3428
}
```

**Low-confidence human (casual ramen review):**

## Transparency Label Variants

All three variants are returned in the `label` field of `POST /submit`.

**High-confidence AI (confidence ≥ 0.65):**
> "⚠️ AI-Generated Content Detected — Our analysis suggests this content was likely generated by an AI writing tool (confidence: 68%). This label is based on automated detection and may not be perfect. If you believe this is incorrect, you can submit an appeal."

**Uncertain (confidence 0.40–0.64):**
> "🔍 Attribution Unclear — Our analysis was unable to determine with confidence whether this content was written by a human or generated by an AI tool (confidence: 65%). We've flagged it for transparency. If you are the human author, you can submit an appeal to provide context."

**High-confidence Human (confidence < 0.40):**
> "✅ Likely Human-Written — Our analysis suggests this content was written by a human (confidence: 15%). Attribution signals did not detect significant markers of AI-generated text."

## Rate Limiting

Applied to `POST /submit`:
- **10 requests per minute**
- **100 requests per day**

**Reasoning:** A real creator submitting their own work would rarely submit more than a few pieces per session. 10/minute is generous for legitimate use while blocking scripts that flood the system. 100/day caps automated abuse across a full day. ## Appeals Workflow

Creators can contest a classification via `POST /appeal`:

```json
{
  "content_id": "5a4346d5-5197-41cd-b87a-e9aba68749cb",
  "creator_reasoning": "I wrote this myself. I am a non-native English speaker and my writing style may appear more formal than typical."
}
```

Response:
```json
{
  "message": "Appeal received successfully.",
  "content_id": "5a4346d5-5197-41cd-b87a-e9aba68749cb",
  "status": "under_review"
}
```

The audit log entry is updated with `status: under_review` and the `appeal_reasoning` field populated. A human reviewer can access the full entry via `GET /log`.

## Known Limitations

**Formal human academic writing:** A well-structured essay with consistent paragraph length and formal vocabulary may score high on stylometrics (low variance = AI-like), even though it is genuinely human. This is a direct consequence of how sentence length variance and TTR are computed — they penalize consistency, which is a property of skilled academic writing, not just AI. The LLM signal partially corrects this, but it is not a complete fix.

**Short creative writing:** Texts under ~50 words give the stylometric signal almost no data — sentence length variance and TTR are statistically meaningless at that length. The system will produce `uncertain` for most short poems or haiku regardless of origin, which is the honest answer but not very useful.

## Spec Reflection

**Where the spec helped:** Writing out the three label variants in `planning.md` before coding made the `labeler.py` implementation trivial — the text was already decided, so the code was just mapping thresholds to pre-written strings. Without that step I would have made up label text on the fly and likely produced inconsistent wording.

**Where implementation diverged:** The spec described the stylometric signal as combining sentence length variance, TTR, and punctuation density with equal weight. During testing I found punctuation density was noisy and unreliable on short texts, so I reduced its weight to 15% (from a notional 33%) and increased sentence variance to 45% and TTR to 40%. The signal now performs more consistently on the test inputs from the spec.

## AI Usage

**Instance 1 — Flask app skeleton and detector function:** I provided the Architecture section from `planning.md` and the Detection Signals section and asked Claude to generate the Flask app skeleton with a `POST /submit` stub and the `detect_with_llm()` function. The generated function returned raw text instead of a guaranteed float, so I added the `try/except ValueError` block with a `0.5` fallback and clamped the output to `[0.0, 1.0]` with `max(0.0, min(1.0, score))`.

**Instance 2 — Stylometric scoring weights:** After implementing `stylometrics.py`, I tested it on all four spec inputs and found punctuation density was producing inconsistent results on short texts. I described the problem to Claude and asked it to suggest revised weights. It proposed reducing punctuation density weight to 15% and redistributing to sentence variance (45%) and TTR (40%). I tested this revised weighting against all four inputs and confirmed it produced more consistent separation between clearly AI and clearly human text before committing it.
